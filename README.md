# いらないアセット消しちゃうもんネーター

VRChat のアバター改変用 Unity プロジェクトから、**どのアバターからも参照されていない未使用アセット**（買ったまま着せていない衣装、試して外した髪型、使わなかったテクスチャなど）を検出し、**安全に退避**してディスク容量を取り戻す Unity Editor 拡張です。

> **削除ではなく「退避」が既定です。** 退避したアセットはプロジェクトルート直下のフォルダへ移動されるだけで、いつでも元に戻せます。誤検出でデータを失わないことを最優先に設計しています。

---

## 動作環境

| 項目 | 要件 |
|---|---|
| Unity | 2022.3 (LTS) |
| VRChat SDK | Avatars 3.7.0 以上（アバターの自動検出に使用。無くても手動指定で動作） |
| OS | 全機能: Windows。キャッシュ掃除機能のみ Windows 専用 |

---

## インストール（VPM）

1. VCC / ALCOM にリポジトリを追加します。
2. プロジェクトの Manage Packages から「いらないアセット消しちゃうもんネーター」を追加します。

導入すると Unity のメニューに `Tools > いらないアセット消しちゃうもんネーター` が追加されます。

---

## 基本的な使い方

ツールは `Tools > いらないアセット消しちゃうもんネーター` で開きます。ウィンドウは5つのタブに分かれています。

### 1. ルート (Roots) — 「使っている起点」を教える

このツールは「登録したアバターから辿れる（＝使われている）アセット」以外を未使用候補とみなします。**ここでアバターを取りこぼすと、使用中のアセットを誤って未使用と判定します。** まず最初に設定してください。

- **フォルダを登録**: アバターの `.prefab` / `.unity`（シーン）を含むフォルダを、ドラッグ＆ドロップまたは「フォルダを選択して追加」で登録します。フォルダ内は再帰的に走査されます。
- **個別ファイルを追加**（任意）: 単一の Prefab / シーンだけをルートに加えられます。
- **アバターを自動検出**: `VRCAvatarDescriptor` を持つ Prefab を自動で探します。チェックを外すと個別に除外できます。
- 以下は参照が無くても自動で「使用中」として扱われます: `Resources/` `Editor Default Resources/` `StreamingAssets/` 配下、Build Settings に登録されたシーン、`Assets/` 直下の `.asset`。

> アバターの起点（Prefab / シーン）が1つも無い状態ではスキャンできません（全アセットが削除候補になり危険なため）。

### 2. 保護 (Protection) — 消したくないものを守る

参照グラフで追いきれないツール・シェーダー・コード資産などを、退避対象から自動的に除外します。判定に迷うものは消さずに保護する方針です。

- **自動で保護されるもの**: `.cs` / `.asmdef` / `.dll` / シェーダー（`.shader` 等）/ `package.json` を含むフォルダ、`Editor/` を持つフォルダ、既知ツール名（VRCSDK, lilToon, Poiyomi, Modular Avatar, NDMF, VRCFury など）を含むパス、特定拡張子。
- **ユーザーホワイトリスト**: `Assets/MyTools/**` のような glob パターンで、保護したいフォルダを自分で追加できます。スキャン結果の各行の「保護に追加」からもワンクリックで登録できます。

### 3. スキャン結果 (Scan Result) — 検出して退避する

1. （任意）**スキャン範囲**にフォルダを登録すると、そのフォルダ配下だけが退避候補として出ます。未設定ならプロジェクト全体が対象です。「まず衣装フォルダだけ整理したい」というときに使います。
   範囲を絞っても**参照の解析は常にプロジェクト全体で行います。** 範囲外のアバターから使われているアセットが、範囲を絞ったせいで未使用と判定されることはありません。
2. （任意）判定粒度を設定します。既定は「導入単位フォルダを自動推定」（BOOTH の「1商品＝1フォルダ」に合わせ、フォルダ内に使用中アセットが1つでもあればフォルダごと残します）。上級者向けに**ファイル単位モード**もあります（既定 OFF）。
3. **「スキャン実行（非破壊）」** を押します。スキャン自体は何も動かしません。
4. 結果はサイズの大きい順に並びます。各行にパス・サイズ・種別・判定根拠が表示されます。パスをクリックすると Project ウィンドウで対象を確認できます。
5. 退避したい行に**チェック**を入れます（既定は全選択 OFF。必ず自分で選びます）。
6. （任意）**「退避前に .unitypackage としてバックアップする」** をオンにすると、退避と同時に `.unitypackage` も書き出します（もう一段の保険）。
7. **「選択項目を退避」** を押し、確認ダイアログで実行します。対象はプロジェクトルート直下の `_UnusedAssets_日時/` へ、`.meta` ごと移動されます。

退避した結果、**中身が空になった親フォルダは `Assets/` に残しません。** `Assets/` 直下が空の抜け殻で散らかるのを防ぐためです。畳んだフォルダも退避フォルダへ送られるので、復元すれば元どおり（フォルダの GUID も保たれます）。畳まれる数は退避前の確認ダイアログに表示されます。

**退避後は Console に Missing 参照の警告が出ていないか確認してください。** 問題があれば下記の復元で戻せます。

#### 退避済みフォルダの復元・完全削除

プロジェクト直下の `_UnusedAssets_…` フォルダは**自動で検出され、一覧表示されます。** スキャンを実行しなくても、退避日時・件数・サイズを見ながら操作できます（退避したまま Unity を再起動しても、ここから戻せます）。

- **復元**: 元の場所へ戻します。`.meta` ごと戻すので参照も復活します。
- **完全削除**（取り消し不可）: 中身が本当に不要と確認できたら、退避フォルダを完全に削除できます。二段階の確認があり、本ツールが作った退避フォルダ以外は削除できません。

退避フォルダを別の場所へ移動した場合は、「退避フォルダを選んで復元」から手動で選ぶこともできます。

### 4. 重複検出 (Duplicates) — 内容が同じファイルを探す

`Assets/` 配下を SHA-256 で照合し、**内容が完全に一致するファイル**を検出します。複数の配布物が同じシェーダーやテクスチャを同梱している場合などに見つかります。

- これは**検出（レポート）のみ**で、自動では何も移動・削除しません。各ファイルが使用中か未参照かも表示します。
- 参照されている重複を消すと壊れるため、内容を確認したうえで手動で判断してください。

### 5. キャッシュ掃除 (Cache) — Library / Temp を消して容量を取り戻す（Windows 専用）

アセットを退避しても `Library/` 配下のキャッシュは縮みません。ここを全削除すると容量を大きく取り戻せる場合があります。**ただしデータではなく「時間」を失う操作です。**

> ⚠️ 削除後、**次回 Unity 起動時に全アセットの再インポートが発生し、プロジェクト規模により数分〜数十分（それ以上のことも）かかります。** Assets が大きいプロジェクトほど時間がかかります。「一瞬で終わる」操作ではありません。

Unity 実行中は `Library/` を直接消せないため、次の流れになります。

1. **「サイズを計測 / 更新」** で現在の Library / Assets の規模を確認します。
2. **「次回 Unity 終了時にキャッシュを掃除する（予約）」** を押し、二段階の確認（「時間がかかることを理解しました」チェックを含む）を経て予約します。
3. 通常どおり Unity を終了すると、外部ヘルパーが Unity の終了を待ってから `Library/` `Temp/` などを削除します。手作業は不要です。
4. 次回起動時に再インポートが実行され、完了時間の目安が記録されます（次回以降の見積りに使われます）。

- **今すぐ実行**: 「今すぐ Unity を終了して掃除する」で、確認後すぐに終了＆掃除します。
- **フォールバック**: うまく動かない環境向けに `clean-cache.bat` を生成できます。Unity を完全に終了してから手動実行してください。
- 掃除対象は `Library/` `Temp/` `obj/` `.vs/`（任意で `Logs/`）と `*.csproj` / `*.sln`。**`Assets/` `Packages/` `ProjectSettings/` には一切触れません。**

**推奨タイミング**: 作業終了時・離席前・就寝前など、しばらく Unity を使わないときに実行してください。

## 安全のための設計

- **既定は退避（移動）で、削除ではありません。** `.meta` ごと移動するので、戻せば参照も復活します。
- **迷ったら保護。** 消し損ね（未使用なのに残る）は無害ですが、誤削除は購入資産を壊します。判定に迷うものは保護側に倒します。
- **スキャンは常に非破壊。** 実際にファイルが動くのは、明示的に退避ボタンを押したときだけです。
- 文字列パスやコード経由の参照など、依存関係の解析で見えない参照もあり得ます。だからこそ「削除」ではなく「退避」を既定にしています。
