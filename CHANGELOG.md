# Changelog

このプロジェクトの変更履歴。バージョニングは [Semantic Versioning](https://semver.org/lang/ja/) に従う。

## [0.3.1] - 2026-07-15

未使用のはずのアセットが「なぜ候補に出てこないのか」を追えるようにし、衣装変換ツールなどの生成物を掃除できるようにした。

### Added
- **保護された単位の一覧表示**: 保護されて候補から外れたフォルダを、**理由つきで**一覧表示するようになった（既定は折り畳み）。従来は「保護 N 件」という件数しか出ず、どのフォルダがなぜ除外されたのか分からなかった。**候補が 0 件のときにも表示される。**「範囲を絞ったのに何も出てこない」というときに、理由がその場で分かる。
- **暗黙ルートの一覧表示と個別除外**: 参照が無くても起点として扱われるアセット（`Resources/` 配下、`Assets/` 直下の `.asset` など）を一覧表示し、チェックを外して起点から除外できるようになった。ツールの設定アセットが「最後に変換した衣装」などを掴んでいると、その衣装がいつまでも「使用中」と判定されるため、それを解消できる。既定はすべて起点のままで、挙動は変わらない。
- **ツール出力フォルダの登録**: ツール本体（スクリプトや DLL）と生成物が同じフォルダに同居している場合、生成物が未使用でも巻き添えで保護され、候補に出なかった。出力フォルダを登録すると、その中身をツール本体と切り離した判定単位として扱い、未使用の生成物を候補に出せるようになる。
  **保護を外す設定ではない。** ツール本体は登録後も保護されたままで、出力フォルダの中にスクリプトやシェーダーが含まれていれば、それも従来どおり保護される。
- **候補のグループ表示と連動選択**: 互いに参照し合っている候補をひとまとまりとして表示し、まとめて選択できるようになった。生成物が変換元のマテリアルを共有しているような場合に、**一部だけを退避して残りが参照切れになるのを防ぐ。**

### Changed
- **設定の保存先を `Assets/zzz_pytr/Asset-Keshichaumon-Nator/Settings.asset` に移した。** 作者名フォルダの下に置き、めったに開かない設定フォルダが Project ウィンドウの先頭に居座らないよう、末尾側に並ぶ名前にしている。
  **旧パス（`Assets/Asset-Keshichaumon-Nator/`）の設定は初回起動時に自動で移行される。** GUID を保ったまま移動するため、登録内容は失われない。空になった旧フォルダは自動で削除される。移行に失敗した場合は、空の設定で上書きせず旧パスの設定を使い続け、警告をログに出す。
- 登録済みフォルダの一覧を枠で囲い、件数を表示し、登録順に並べるようにした。フォルダ一覧が3つに増えて見分けづらくなっていたため。

## [0.3.0] - 2026-07-14

### Added
- **スキャン範囲の限定**: 未使用アセットのスキャンで、対象フォルダを指定できるようになった。指定したフォルダ配下だけが退避候補として列挙される（未設定ならプロジェクト全体）。範囲を絞っても参照の解析は常にプロジェクト全体で行うため、範囲外のアバターから参照されているアセットが未使用と誤判定されることはない。
- **空になった親フォルダの畳み込み**: 退避によって中身が空になった親フォルダを `Assets/` に残さないようにした。畳んだフォルダも `.meta` ごと退避フォルダへ送られるため、復元すればフォルダの GUID を保ったまま元に戻る。畳まれる数は退避前の確認ダイアログに表示される。
- **退避済みフォルダの一覧表示**: プロジェクト直下の `_UnusedAssets_…` を自動検出して一覧表示するようにした。**スキャンを実行しなくても**復元・完全削除ができる（退避したまま Unity を再起動した場合でも操作できる）。一覧に出るのは本ツールが作った退避フォルダのみ。

### Changed
- 内部の略称を `AKM` から `AKN` に統一した（名前空間・アセンブリ名・退避マニフェストのファイル名・キャッシュ掃除用ディレクトリ名）。保存済みの設定はそのまま引き継がれる。

### ⚠️ 破壊的変更
- 退避フォルダ内のマッピングファイル名が `.akm-relocation.json` から `.akn-relocation.json` に変わった。このため **0.2.0 以前で作成した退避フォルダ（`_UnusedAssets_…`）は、本バージョンのツールからは復元・完全削除ができない。**
- **アップグレード前に、既存の退避フォルダを 0.2.0 のツールで復元しておくことを推奨する。** 復元し損ねた場合も、退避フォルダ内はディレクトリ構造がそのまま保持されているため、手動で `Assets/` 配下へ戻せる（`.meta` も同梱されているので GUID は失われない）。

## [0.2.0] - 2026-07-13

### Added
- ビルドキャッシュ掃除機能（`Library/` `Temp/` などの全削除。Windows のみ）。Unity 終了時に外部ヘルパーが削除する予約方式で、手作業なしに掃除できる。目立つ警告・プロジェクト規模の数値提示・削除対象の全件列挙・二段階確認・終了時の再警告・起動時の再インポート通知・所要時間の実測記録・手動実行用 `clean-cache.bat` を同梱。
- 重複アセット検出（SHA-256 で内容が完全一致するファイルを検出。レポートのみで自動削除はしない）。
- 退避前の `.unitypackage` バックアップ（任意）。
- 退避フォルダの完全削除（二段階確認つき。退避フォルダ以外は削除しない）。
- ファイル単位モード（フォルダ集約を無効化して1ファイルずつ列挙。上級者向け・既定 OFF）。
- 依存グラフのキャッシュによるスキャン高速化。

### Changed
- README を使い方ガイドとして全面刷新。
- ユーザーに見える文字列・コード内から内部設計資料の識別子を除去し、平易な表現へ統一。

## [0.1.0] - 2026-07-13

### Added
- 初回リリース（Phase 1 / MVP）。
- アバターから参照されていない未使用アセットの検出（Mark & Sweep）。
- ルート集合の指定: フォルダ登録・個別ファイル・`VRCAvatarDescriptor` 自動検出・暗黙ルート（Resources 等）。
- 保護ルール: 構造ヒューリスティック・既知ツール名・拡張子・ユーザーホワイトリスト（glob）。
- 導入単位フォルダ判定（固定深度／自動推定）。
- 退避（`.meta` 同伴・プロジェクトルート直下へ移動）と復元。
- 3タブ構成の Editor ウィンドウ、確認ダイアログ、進捗表示。
